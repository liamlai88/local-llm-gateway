# RAG vs 纯 LLM 实证报告

> **核心问题**：检索增强 (RAG) 能为小模型带来多大提升？延迟和成本代价是多少？  
> **实验时间**：2026.04  
> **实验工具**：本地 Qwen2.5-1.5B Q4 + 百炼 text-embedding-v2 + ChromaDB

## 性能与成本分析

### 延迟对比

| 阶段 | 纯 LLM | RAG (首次) | RAG (常态) |
|------|--------|------------|------------|
| 检索 | - | 1330ms | ~320ms |
| 生成 | ~500ms | ~600ms | ~500ms |
| **总计** | **~500ms** | **~1900ms** | **~820ms** |

**洞察**：首次检索延迟是常态的 4 倍 —— 百炼 embedding API 冷启动效应。

## 三个反直觉洞察

### 洞察 1：拒答 ≠ 安全，幻觉才是杀手

**Q3 的纯 LLM 输出片段**：

```
"RAG（Recommender Application for Users and Groups）是一种推荐系统，用于向用户或组成员
提供个性化的产品、服务或其他资源的推荐。为了有效地部署RAG应用，您需要考虑以下几点：
1. 硬件要求：通常，RAG的后台服务器会运行大量的计算任务...
2. 基础技术平台：可使用 Apache Spark、Hadoop..."
```

**问题**：模型自信地**编造了一个错误的 RAG 全称**，然后基于这个错误前提，给出了一整套 Spark/K8s 的伪专业回答。

这是 LLM 最危险的幻觉模式——**错得理直气壮**。客户如果不懂行，会照着错误方案去做。

**SA 视角**：
> "向客户解释 RAG 价值时，不要只说'解决知识缺失'，更要强调'消除伪知识冒充真知识'。Q3 这个案例就是最直观的证据——拒答的客户体验差但安全，瞎编的客户体验好但致命。"

---

### 洞察 2：RAG 不一定更贵（Q3 反例）

直觉认为 RAG 增加了输入 token，必然更贵。但 Q3 显示 **RAG 反而便宜了 2%**。

**原因**：纯 LLM 不知道答案时会**用大段文字"装懂"**，输出 token 暴涨；RAG 知道答案就直接答 "Dify" 一个词。

**生产价值**：
- RAG 不仅提升准确率，还压制了"啰嗦输出"
- 在客服等场景，RAG 能让回答**更精准+更短**，双向降本

---

### 洞察 3：首次检索延迟 = 4× 常态

```
第1次检索: 1330ms  ← Embedding API 冷启动
第2-4次:    300-350ms ← 稳态
```

**生产建议**：
1. **服务启动后做 embedding 预热**（发一个 dummy 查询）
2. **高频问题缓存向量**（不只缓存答案，缓存向量本身）
3. **本地部署 bge-m3 模型**绕开 API 调用延迟

## 商业决策模型

### 场景：企业内部知识库 SaaS（员工自助问答）

| 方案 | 准确率 | 月成本（10万次问答） | 推荐度 |
|------|--------|----------------------|--------|
| 纯 1.5B 部署 | ~0% | ¥7 | ❌ 业务不可用 |
| **1.5B + RAG (本方案)** | **~95%+** | **¥13** | **⭐⭐⭐⭐⭐ 最优** |
| 7B 模型 + RAG | ~98% | ¥80 | ⭐⭐⭐⭐ 高质量 |

**关键洞察**：**RAG 把 1.5B 小模型从"业务不可用"变成"主力可选方案"**，多花 ¥6/月换来从 0% 到 95% 的准确率跃升。这是企业知识库场景最大的成本杠杆。

## RAG 架构设计要点（本实验验证）

```
[用户问题]
    │
    ▼
百炼 text-embedding-v2 (1536维)
    │
    ▼
ChromaDB 余弦相似度 Top-K
    │
    ▼ Top-2 文档片段
[拼接 Prompt]
    │
    ▼
Qwen2.5-1.5B Q4 (本地)
    │
    ▼
[带溯源的答案]
```

### 关键参数选择
- **Chunking**: 300 字 + 50 字 overlap（按段落优先，超长按句号切）
- **Top-K**: 2-3（实验中 Top-2 已足够）
- **Embedding 模型**: 百炼 text-embedding-v2（中文场景最优）
- **Vector Store**: ChromaDB（嵌入式，无需独立服务）
